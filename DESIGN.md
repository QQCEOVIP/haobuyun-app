# 云号簿 - 设计规范

## 产品定位
**通讯录健康度管理工具** - 帮助用户识别失效号码，避免打扰

## 设计风格
**柔和卡片风 + 工具实用性**
- 新拟态双层阴影 + 暖灰白统一底色
- 简洁实用优先，减少视觉干扰
- 数据可视化突出（健康度仪表盘）

## 配色方案

### 主配色
- **主色**：#4A90D9（科技蓝，传达信任感）
- **辅色**：#67C23A（绿色，活跃/安全）
- **警告色**：#E6A23C（橙色，可能失效）
- **危险色**：#F56C6C（红色，确定失效）

### 背景色
- **背景**：#F5F7FA（浅灰白）
- **卡片**：#FFFFFF
- **凹陷**：#EBEEF5

### 文字色
- **主文字**：#303133
- **次文字**：#909399
- **占位符**：#C0C4CC

## 页面结构

### 1. 底部 Tab 导航
- 首页（健康度概览）
- 通讯录（联系人列表）
- 清理（失效号码清理）
- 我的（个人中心）

### 2. 首页
- **健康度仪表盘**：圆形进度环显示整体健康度
- **统计数据**：总数/活跃/可能失效/确定失效
- **快捷入口**：一键检测、批量清理、备份通讯录
- **最近联系人**：展示最近活跃的联系人

### 3. 通讯录页
- **搜索栏**：支持姓名/号码搜索
- **筛选标签**：全部/活跃/可能失效/确定失效
- **列表展示**：头像+姓名+号码+状态标签+最后联系时间
- **分组管理**：支持按标签分组

### 4. 清理页
- **失效号码列表**：按失效概率排序
- **批量选择**：支持多选确认删除
- **备份提醒**：清理前自动提示备份

### 5. 联系人详情页
- **基础信息**：头像、姓名、号码
- **号码状态**：状态标签 + 判断依据
- **标签管理**：自定义标签
- **操作按钮**：标记失效、编辑、删除

### 6. 我的页
- **用户信息**：头像、昵称
- **功能入口**：备份通讯录、导出通讯录、标签管理
- **设置**：隐私协议、用户协议、关于我们

### 7. 登录页（首次启动/未登录）
- **邮箱密码登录**
- **应用图标和名称展示**
- **隐私协议勾选**

## 数据模型

### 联系人 (contacts)
- id, user_id, name, phone, avatar_url
- status: 'active' | 'maybe_invalid' | 'invalid' | 'unknown'
- invalid_reason: string | null
- last_contact_date: timestamp
- created_at, updated_at

### 标签 (tags)
- id, user_id, name, color, icon
- created_at

### 联系人标签关联 (contact_tags)
- contact_id, tag_id

### 失效标记上报 (invalid_reports)
- phone_hash (脱敏), report_count, report_type
- created_at, updated_at

### 备份记录 (backups)
- id, user_id, contact_count, backup_url
- created_at

## 技术实现

### 前端
- Expo SDK 54 + React Native
- expo-contacts 读取通讯录
- expo-linking 深链接
- @react-native-async-storage/async-storage 本地存储
- Supabase Auth 登录

### 后端
- Express.js API
- Supabase Database 数据存储
- 众包标记数据存储（脱敏）

## 隐私合规
- 通讯录数据仅本地处理
- 失效标记仅上传手机号哈希
- 提供完整隐私协议和用户协议
- 不发短信、不拨号、不调用运营商接口
